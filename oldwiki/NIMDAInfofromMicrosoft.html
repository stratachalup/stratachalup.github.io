Subject: Virus Alert: W32.Nimda.A@mm Worm Virus

 this message with the internal aliases.  Thanks!




Microsoft is investigating reports of a new virus that is

 affecting many of our customers.  The investigation is 


ongoing, and the information we have at this point is preliminary. We will update it as we learn more.


It appears that the virus spreads via three methods: by sending itself
via email, by infecting vulnerable web servers, and through file shares.


Email:  The virus includes itself as a file attached to an email.    The
worm uses MAPI calls to read email in your email program's inbox in
order to find new addresses.  Note that the worm uses its own SMTP
engine to send itself as an attachment with no body and with random
subject lines.  When a user receives this email, the readme.exe poses as
an audio .wav file.  What you and your customer should be doing to
thwart this:



    

  •        Apply MS01-020 as the email variant uses this vulnerability as soon as the user opens the mail.  Please note that Outlook is not being used to spread the email variant so even though the OESU is a terrific idea, it is not pertinent to this situation.  

  •        IE 5.01 Service Pack 2 includes the fix in MS01-020 

  •        IE 5.5 Service Pack 2 includes the fix in MS01-020 

  •        IE6 (not vulnerable) 

  •        Don't open untrusted attachments.

    


You might also consider unregistering .eml file types.


Web servers:  Infected machines try to spread the infection to web servers via several known vulnerabilities affecting IIS 4.0 and 5.0. All of these vulnerabilities are eliminated by the most recent patch for IIS, which was delivered over a month ago via Microsoft Security Bulletin MS01-044.  Please note that if you did not clean up infection from Code Red, you may be vulnerable to this virus regardless of the patches applied!


File shares:  Infected machines try to find neighboring computers that have been configured to share files.  If it finds a computer that gives it write access, it inserts infected files onto the machine.  No Windows systems perform file sharing by default.  Any infected machine which has mapped network drives will likely infect all of the files on the mapped drive and its subdirectories.  What you might do to thwart this:



    

  •        Use the CALCS program to modify the permissions on TFTP.EXE to remove all use;

    
        CALCS %systemroot%/system32/tftp.exe /D Everyone

    
        CALCS %systemroot%/system32/tftp.exe /D System

    

  •        Don't let TFTP out through the network gateway because newly infected machines may try and TFTP internally from some other infected machine you have on the network.

    

    


As with any virus, each customer must contact their AV vendor and work closely with them for the updated definitions file as well as information on cleaning infected systems.
